# ContextRecoveryHook

> **PreCompact backup + SessionStart recovery — the lightest Claude Code context preservation plugin.**

ContextRecoveryHook prevents **context amnesia** during Claude Code's auto-compaction. It works in two phases:

```
对话进行中 → [自动压缩触发] → PreCompact: 备份 transcript + 生成 CONTEXT.md
           → 压缩发生
下次启动 → SessionStart: 读取 CONTEXT.md → 注入 additionalContext
```

No mono's 23 commands. No wiki links. Just **2 hooks + 2 context files**.

---

## Features

| Feature | Description |
|---------|-------------|
| **Auto-backup** | Transcript backed up to `~/.claude/logs/transcript_backups/` before every auto-compaction |
| **Context generation** | Extracts recent prompts and file paths into `~/.claude/CONTEXT.md` |
| **Session recovery** | SessionStart hook injects saved context as `additionalContext` on resume |
| **TODO tracking** | Timestamps `~/.claude/TODO.md` for work continuity |
| **Event logging** | All events logged to `~/.claude/logs/events.json` |
| **Manual commands** | `/context-save` and `/context-restore` slash commands |

## Installation

### Option 1: Claude Code Plugin (recommended)

```bash
# Clone the repo
git clone https://github.com/wxhou/context-recovery.git

# Run install script (installs to current directory's .claude/)
cd context-recovery
bash install.sh

# Or install globally (home directory)
bash install.sh ~
```

### Option 2: Manual Installation

```bash
# Copy hook handlers to your .claude directory
cp hooks-handlers/pre_compact.py ~/.claude/hooks-handlers/
cp hooks-handlers/session_start.py ~/.claude/hooks-handlers/
chmod +x ~/.claude/hooks-handlers/*.py

# Copy slash commands
cp commands/*.md ~/.claude/commands/

# Add hooks to ~/.claude/settings.local.json
# (see hooks/hooks.json for the configuration)

# Create context files
touch ~/.claude/CONTEXT.md
touch ~/.claude/TODO.md

# Restart Claude Code
```

## How It Works

### Phase 1: PreCompact (Before Compaction)

When Claude Code's context is 70% full and auto-compaction triggers:

1. `pre_compact.py` reads the full transcript
2. Backs up transcript to `~/.claude/logs/transcript_backups/transcript_auto_YYYYMMDD_HHMMSS.jsonl`
3. Extracts recent prompts + file paths → writes `~/.claude/CONTEXT.md`
4. Updates timestamp on `~/.claude/TODO.md`
5. Logs event to `~/.claude/logs/events.json`

### Phase 2: SessionStart (On Resume)

When you resume the session (or start a new one):

1. `session_start.py` checks for existing `CONTEXT.md` and `TODO.md`
2. Loads recent transcript backup (if < 7 days old)
3. Builds an `additionalContext` block with:
   - Previous session context
   - Active TODO items
   - Recent conversation snippets
   - Git status
4. Injects it as structured context for Claude to read

## Configuration

### Hook Matcher

By default, PreCompact fires on **both** manual (`/compact`) and automatic compaction:

```json
"PreCompact": [{
  "matcher": "auto",   // "auto" = only auto-compact, "" = both
  "hooks": [{ "type": "command", ... }]
}]
```

To only backup on auto-compaction (recommended):
```json
"PreCompact": [{ "matcher": "auto", ... }]
```

### Requirements

- Python 3.11+
- [uv](https://astral.sh/uv) (auto-installed by script)

## Context Files

### CONTEXT.md

Auto-generated before each compaction. **Do not edit manually** — it's overwritten.

```markdown
# Claude Code Session Context
> Auto-generated by ContextRecoveryHook

## Session Info
- Session ID: `abc123...`
- Generated: 2026-03-25 16:00:00

## Recent Work
### Recent User Requests
- Fix the login bug in auth.py

### Files Recently Touched
`auth.py`, `session.py`, `config.json`

## Recovery Notes
_Add your notes here before the next session..._
```

### TODO.md

**Manually maintained.** Add your active work items:

```markdown
# TODO — Work Items

- [ ] Fix authentication flow
- [ ] Write tests for user model
- [x] Set up database schema
```
